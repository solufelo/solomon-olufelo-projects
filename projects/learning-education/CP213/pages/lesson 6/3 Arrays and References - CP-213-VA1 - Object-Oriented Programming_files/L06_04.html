3 Arrays and References

Arrays are objects, and array types are (usually) considered to be class types. Like class types, a variable of an array type holds a reference. That is,a variable of an array type holds the address of where the array object is stored in the memory.

An array can be viewed as a collection of indexed variables or as a single item whose value is a collection of values of a base type.

An array variable names the array as a single item: double[] a;

A new expression creates an array object and stores the object in the memory: new double[10].

An assignment statement places a reference to the memory address of an array object in the array variable: a = new double[10];

The previous steps can be combined into one statement


	double[] a = new double[10];

Note: the new expression that creates an array invokes a constructor that uses a nonstandard syntax and as a result of the assignment statement above, an array contains a single value:  a memory address or reference.

Since an array is a class type, the behavior of arrays concerning the assignment (=), equality testing (==), and parameter passing are the same as that described for classes, lesson 05.  

3.1 Arrays of Objects

The base type of an array can be a class type:


	Date[] holidayList = new Date[20];

The example above creates 20 indexed variables of the type Date. It does not create 20 objects of the class Date. Each of these indexed variables is automatically initialized to null. Any attempt to reference any of them at this point would result in a "null pointer exception" error message.

Like any other object, each of the indexed variables requires a separate invocation of a constructor using new (singly, or perhaps using a for loop) to create an object to reference. 


	holidayList[0] = new Date();
            . . .
	holidayList[19] = new Date();

OR


	for (int i = 0; i < holidayList.length; i++)
		holidayList[i] = new Date();

Each of the indexed variables can now be referenced since each holds the memory address of a Date object.

3.2 Array Parameters

Both arrays indexed variables and entire arrays can be used as arguments to methods. An indexed variable can be an argument to a method in the same way that any variable of the array base type can be an argument. See the example below.


	double n = 0.0;
	double[] a = new double[10];//all elements are initialized to 0.0

	int i = 3;

Given myMethod, which takes one argument of type double, all of the following are legal:


	myMethod(n);//n evaluates to 0.0
	myMethod(a[3]);//a[3] evaluates to 0.0
	myMethod(a[i]);//i evaluates to 3, a[3] evaluates to 0.0

An argument to a method may be an entire array. Array arguments behave like objects of a class. Therefore, a method can change the values stored in the indexed variables of an array argument.

A method with an array parameter must specify the base type of the array only. It does not specify the length of the array.

The following method, doubleElements, specify an array of double as its single parameter a:


	public class SampleClass {
	   public static void doubleElements(double[] a){
	    int i;
	    for (i = 0; i < a.length; i++)
	      a[i] = a[i]*2;
	    . . .
	  }...
	}

Arrays of double may be defined as follows:


	double[] a = new double[10];
	double[] b = new double[30];

Given the arrays above, the method doubleElements, from class SampleClass, can be invoked as follows:


	SampleClass.doubleElements(a);
	SampleClass.doubleElements(b);

Note: no square brackets are used when an entire array is given as an argument, and a method that specifies an array for a parameter can take an array of any length as an argument.

3.3 Privacy Leaks with Array

If an accessor method does return the contents of an array, special care must be taken just as when an accessor returns a reference to any private object


	public double[] getArray() {
		return anArray;  //BAD CODE!
	}

The example above will result in a privacy leak. The accessor method would simply return a reference to the array anArray itself. Instead, an accessor method should return a reference to a deep copy of the private array object.

In the code snippet below, a deep copy of an array is created where a and count are instance variables of the class containing the getArray method:


		public double[] getArray() {
		  double[] temp = new double[count];

		  for (int i = 0; i < count; i++)
		    temp[i] = a[i];
		  return temp
		}

If a private instance variable is an array that has a class as its base type, then copies must be made of each class object in the array when the array is copied:


	public ClassType[] getArray() {
		ClassType[] temp = new ClassType[count];
		for (int i = 0; i < count; i++)
          // a copy is made for each object element
		    temp[i] = new ClassType(someArray[i]); 		  
  		return temp;
  	}

3.4 Array as Method Return Type

In Java, a method may also return an array. The return type is specified in the same way that an array parameter is specified. In the example below, an array is used as a return type. 


	public static int[] incrementArray(int[] a, int increment) {
		
		int[] temp = new int[a.length];
		int i;
		for (i = 0; i < a.length; i++)
			temp[i] = a[i] + increment;
		return temp;
}

Note: when an instance variable names an array, it is not always necessary to provide an accessor method that returns the contents of the entire array. Instead, other accessor methods that return a variety of information about the array and its elements may be sufficient, and accessor methods donâ€™t need simply return instance variables.

3.5  Use of = and == with Arrays

Because an array variable contains the memory address of the array it names, the assignment operator (=) only copies this memory address. It does not copy the values of each indexed variable. Using the assignment operator will give two array variables different names for the same array. In the example below, 


	int a = new int [5] ; 
	int b = new int [6]; 
	b= a;

The memory address in b is now the same as the memory address in a: they reference the same array.

A for loop is usually used to make two different arrays have the same values in each indexed position. Here is an example: 


	int i;
		for (i = 0; (i < a.length) && (i < b.length); i++)
			b[i] = a[i];

Note, the code above will not make b an exact copy of a unless a and b have the same length.

The equality operator (==) only tests two arrays to see if they are stored in the same location in the computer's memory. It does not test two arrays to see if they contain the same values. For example, the result of the boolean expression (a == b) will be true if a and b share the same memory address (and, therefore, reference the same array) and false otherwise.

3.6 Array equals method

In the same way that an equals method can be defined for a class, an equalsArray method can be defined for a type of array.

The following method tests two integer arrays to see if they contain the same integer values. This is how two arrays must be tested to see if they contain the same elements. 


	public static boolean equalsArray(int[] a, int[] b) {
		if (a.length != b.length)
			return false;
		else {
			int i = 0;
			while (i < a.length) {
				if (a[i] != b[i])
					return false;
				i++;
			}
		}
		return true;
	}
