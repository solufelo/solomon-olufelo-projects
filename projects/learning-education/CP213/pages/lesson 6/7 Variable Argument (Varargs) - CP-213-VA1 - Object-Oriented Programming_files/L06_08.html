7 Variable Argument (Varargs)

Starting with Java 5.0, methods taken any number of arguments can be defined. Essentially, it is implemented by taking in an array as an argument, but the job of placing values in the array is done automatically.

The varargs allows the method to accept zero or multiple arguments. It is a better way to do method overloading, you write less and more clear code than overloading and achieve the same result. Less and clear coding eases the code maintenance as well.

If we do not know how many arguments we will have to pass in the method, varargs needs to be followed to avoid maintenance problems by writing less code.

The values for the array are given as arguments. Java automatically creates an array and places the arguments in the array. Note that arguments corresponding to regular parameters are handled in the usual way.

A method that uses varargs has as the last item on its parameter list, a vararg specification of the form: Type... ArrayName.

The method signtuere would be as follows:


	returnType methodName (dataType... variableName)

The three dots called an ellipsis must be included as part of the vararg specification syntax. Following the arguments for regular parameters are any number of arguments of the type given in the vararg specification.

7.1  Varargs Example

Here is an example of how to use varargs in your code.


package lesson06;

public class VariableArgumentExample {

	public static void printLength(String... variable) {
		if (variable.length <= 0) {
			System.out.println(" 0 parameter passed");
		} else {
			System.out.println(variable.length + " parameters are
                         passed" + " and they are: ");
			for (String s : variable) {
				System.out.print(s + " ");
			}
			System.out.println();
		}
	}
	public static void main(String args[]) {
//same print method used over and over with need for method
  overloading  
	 VariableArgumentExample.printLength();
	 VariableArgumentExample.printLength("amazing coding");
	 VariableArgumentExample.printLength("1", "2");
	 VariableArgumentExample.printLength("1", "2", "3");
	 VariableArgumentExample.printLength("1", "2", "3", "4");
	 VariableArgumentExample.printLength("1", "2", "3", "4","5");
	 VariableArgumentExample.printLength("1", "2", "3", "4", 
                                                          "5","6");
	 VariableArgumentExample.printLength("have you noticed the power of varargs?");
	}
}

7.3 Rules to Use Varargs

There are rules you must follow when using varargs to compile your program successfully.  The rules are:

  • There can be only one variable argument in the method.
  • Variable argument (varargs) must be the last argument.

7.4 Examples of Varargs that will not compile

This method: 


	void methodName (String... a, int... b) {}

generates a compile-time error for having two varargs arguments (a and b) in the method signature. Java allows only one parameter of type Varargs.  

This method, void methodName (int... a, String b) {}, also generates an error. The reason for the error this time is that the vaargs parameter is the first parameter and not the last argument;  That is inside the parenthesis (),  int â€¦ a comes before b which is not allowed.
